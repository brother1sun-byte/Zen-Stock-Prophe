# Zen Stock Prophet Constitution

## 目的
Zen Stock Prophet / AI Investment Simulator は、日本株リサーチにおける一次情報確認、リスク確認、短期売買の検討材料整理を支援するためのアプリです。

本プロジェクトは、売買推奨ではなく、利用者が自分で判断するための材料を高速・再現可能・透明に提示することを目的とします。

## 原則

### 1. 投資助言に見える表現を避ける
- 「買い」「売り」「推奨」「確実」「勝てる」などの断定表現は禁止します。
- 表現は「材料確認」「一次情報確認」「リスク確認」「判断補助」に限定します。
- スコアや期待損益は、最終判断ではなく参考情報として扱います。

### 2. データ出所と鮮度を明示する
- 株価、開示、決算予定、営業日、ニュース、バックテスト結果は、可能な限り出所と取得時刻を表示します。
- 外部API未設定、取得失敗、キャッシュ利用、手動データ、データ欠損は区別して表示します。
- 不確実な情報を確定情報のように表示しません。

### 3. APIキー・認証情報をフロントへ出さない
- EDINET、J-Quants、その他外部APIキーはバックエンドのみで扱います。
- フロントエンドに秘密情報を渡す実装は禁止します。
- `.env` や認証情報はコミット対象外とします。

### 4. 既存設計を壊さず、最小追加で進める
- 既存UI、既存API、既存テストを壊さないことを優先します。
- 新機能は小さく分割し、段階的に実装します。
- 大規模リファクタリングは、明確な必要性と検証手順がある場合のみ行います。

### 5. テスト可能な仕様から実装する
- 実装前に、目的、ユーザー価値、受け入れ条件、非対象範囲を明確にします。
- 仕様、計画、タスク、検証結果を追跡できる状態にします。
- 可能な限り `npm run build`、`npm run typecheck`、`npm run test:e2e`、バックエンドテストで確認します。

## 実装前ゲート
新しい機能を実装する前に、以下を確認します。

- [ ] 目的が明確である
- [ ] 売買推奨に見える表現がない
- [ ] データ出所・鮮度・欠損時の扱いが決まっている
- [ ] フロントに秘密情報を出していない
- [ ] 既存機能への影響範囲が説明できる
- [ ] 受け入れ条件がテスト可能である

## 変更管理
- 仕様変更は `specs/` 配下に残します。
- 実装完了後も、仕様・計画・タスク・検証結果の対応関係を維持します。
- 不明点は推測で埋めず、`[NEEDS CLARIFICATION]` として明示します。
