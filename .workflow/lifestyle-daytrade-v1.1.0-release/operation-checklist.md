# v1.1.0 Lifestyle Daytrade 運用確認手順書

## P7 追加確認: 判断支援UX

生活導線デイトレ画面を開いたら、各モードの詳細を見る前に以下を確認してください。

1. 「今日見るべきポイント」が表示される。
2. 目的、判断範囲、次に確認することが短く読める。
3. 材料、需給、テクニカル、リスクが分かれて表示される。
4. 手動判断前チェックで、一次情報、現在価格、注文上限、撤退ライン、見送り条件を確認できる。
5. 取得不可、推定、キャッシュ、手入力が必要な価格が隠れていない。
6. 投資助言、利益保証、自動売買、実注文、証券会社API連携、外部ログ送信を行わない旨が表示される。

## 4. 毎朝の確認手順

1. Night Scanまたはウォッチリストから候補銘柄を確認する。
2. Morning Gateで手入力価格を入れる。
3. 注文上限との差を確認する。
4. 見送り条件、ギャップ、VWAP、出来高、急変注意を確認する。
5. 証券アプリと一次情報で最終確認する。
6. 必要な場合のみ、利用者が手動で判断する。

## 5. 仕事中の確認手順

1. Work Monitorで保有中銘柄を見る。
2. 利確候補までの距離を見る。
3. 撤退ラインまでの距離を見る。
4. VWAP割れ、出来高急増、急変注意などの危険サインを見る。
5. 必要な場合のみ、利用者が手動で撤退判断する。

## 6. 引け後の確認手順

1. After Close Reviewで取引結果を記録する。
2. エントリー理由を残す。
3. 終了理由を残す。
4. 改善メモを残す。
5. 保存レビュー件数を確認する。
6. 必要に応じてJSONまたはCSVでローカルにエクスポートする。

## 7. エクスポート確認

- 保存レビュー件数が表示されること。
- JSONエクスポートボタンが表示されること。
- CSVエクスポートボタンが表示されること。
- エクスポートはブラウザ内で完結し、外部APIへ送信しないこと。
- 個人ログをGitに含めないこと。

## 8. 禁止事項

- 自動売買に使わない。
- 自動発注しない。
- 自動売却しない。
- 証券会社APIに接続しない。
- 取引ログを外部送信しない。
- 投資助言として扱わない。
- 利益保証として扱わない。

## 9. データ取得不可時の確認

以下の場合でも画面が壊れず、取得不可、推定、判断保留、見送り優先、手入力してください、データ不足などの安全な表示になることを確認する。

- 不正な銘柄コード
- yfinance取得失敗
- J-Quants APIキー未設定
- EDINET APIキー未設定
- バックテスト材料なし
- ウォークフォワード材料なし
- 出来高データ不足
- 価格データ不足
- localStorageにレビューがない
- localStorageに壊れたレビューJSONがある

## 11. 最終注意

このアプリは、忙しい会社員が限られた時間で候補銘柄、注文条件、利確候補、撤退ライン、危険サイン、過去レビュー傾向を確認するためのデイトレード判断支援ツールです。

投資判断を代行せず、実注文、自動売買、自動発注、自動売却、外部ログ送信を行いません。
