# v1.1.0-lifestyle-daytrade

## P7 判断支援UXベンチマーク改善

`v1.1.0-lifestyle-daytrade` 後の改善として、短期売買支援ツール、投資ダッシュボード、金融UIのベストプラクティスを確認し、生活導線デイトレ画面の冒頭に「今日見るべきポイント」を追加しました。

この追加は、取引判断を代行するものではありません。目的、判断範囲、確認材料、データ不足、一次情報確認の必要性を先に示し、Night Scan / Morning Gate / Work Monitor / After Close Review の詳細を読む前に、手動判断の前提を把握できるようにするものです。

追加された整理:

- 材料、需給、テクニカル、リスクを分けた確認カード
- 一次情報、現在価格、注文上限、撤退ライン、見送り条件の手動判断前チェック
- 推定、取得不可、キャッシュ、手入力が必要な価格の明示
- 投資助言、利益保証、自動売買、実注文、証券会社API連携、外部ログ送信を行わない安全境界の再掲

調査サマリーと改善計画は `.workflow/decision-support-ux-benchmark/` に保存しています。

## Zen Loop Desk 安全ワークフロー

Zen Loop Desk は、Minara風の自律型クオンツデスク思想から、Research -> Thesis -> Verification -> Alert -> Review の検証付きループだけを安全に取り込んだ補助UIです。既存データを統合表示する薄いレイヤーであり、取引判断の自動化や外部実行は行いません。

表示する内容:

- 候補ごとの bullish thesis / bearish thesis
- entry条件、invalidation条件、risk/reward短評
- verified candidate gate の状態
- 条件未達時の `research-only` 表示
- no verified candidate 時に候補を無理に作らない理由

verified candidate として扱う条件:

- `tradeReadiness == ready`
- `decisionAudit.verdict == PASS`
- actionable size あり
- cross-engine confirmation が必要な場合は `aligned`

安全境界:

- 自動売買、注文、broker / RPA 連携、外部通知は行いません。
- `alerts=0` または `status=NO_ACTION` を送信や実行に変換しません。
- 検証未達候補を actionable として扱いません。
- 表示内容は調査補助であり、最終判断は利用者が一次情報を確認して手動で行います。
- 実データ精度改善と市場時間中の実データ確認は別タスクです。

## リリース概要

`v1.1.0-lifestyle-daytrade` は、忙しい会社員が帰宅後、翌朝、仕事中、引け後の生活導線に沿って日本株デイトレードの確認材料を短時間で整理するためのリリースです。

このリリースは投資判断を代行するものではありません。候補銘柄、価格ライン、危険サイン、過去レビュー傾向を確認するための手動判断支援ツールです。

## 追加機能

- Night Scan: 帰宅後に翌日の確認候補を整理
- Morning Gate: 翌朝に手動注文候補か、待機か、見送り優先かを短時間で確認
- Work Monitor: 仕事中に保有継続、利確検討、撤退検討の要点を確認
- After Close Review: 引け後に結果、根拠、改善メモを記録
- 手入力価格: 証券アプリ等で確認した価格を入力し、価格ラインとの差を再計算
- 価格ライン表示: 注文上限、利確候補、撤退ライン、支持線、抵抗線などを表示
- レビュー改善ログ: 過去レビューから注意点と改善ヒントを整理
- JSON/CSVエクスポート: After Close Reviewの保存ログをブラウザ内で出力
- 出来高季節性推定: 時間帯別の出来高傾向を参考材料として表示
- スプレッド推定: 流動性と値幅から参考推定を表示
- スマホ表示改善: 390px、430px、768px幅で主要カードを確認

## 使い方

### 帰宅後: Night Scan

翌日の確認候補を絞るために使います。短期強弱、出来高傾向、価格ライン、イベント材料、過熱感、見送り条件を確認します。

### 翌朝: Morning Gate

通勤中など短時間で確認するために使います。現在価格または手入力価格、注文上限との差、ギャップ、VWAP、出来高、寄り付き直後の待機条件を確認します。

### 仕事中: Work Monitor

頻繁に画面を見られない時間帯に使います。利確候補までの距離、撤退ラインまでの距離、VWAP割れ、出来高急増、急変注意を確認します。

### 引け後: After Close Review

取引結果、当初の根拠、実際に効いた根拠、外れた根拠、改善メモを記録します。保存レビューはJSON/CSVでローカルにエクスポートできます。

## 注意点
