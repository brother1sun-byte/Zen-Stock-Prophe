# Zen Stock Prophet Pro

## 無料日本株リサーチ強化計画

GitHub上の日本株データ取得、EDINET/TDnet、J-Quants、テクニカル分析、AI/機械学習、チャートUI、バックテスト系OSSを調査し、無料で構築できる範囲の改善計画を `docs/free-japan-stock-research-plan.md` に整理しています。

今回追加したP0改善:

- データ出所ラベルを日本語で表示します。
- `synthetic` は補完データ、`cache` は一時保存データ、`J-Quants delayed` は遅延データとして警告します。
- 高度分析画面に「無料リサーチ網羅度」を追加し、株価、ランキング、テクニカル、ウォークフォワード、AI検証、開示・材料、財務の取得状況を確認できます。
- 高度分析画面に「開示・決算チェック」を追加し、EDINET、TDnet、J-Quants、キャッシュの取得状況と直近材料イベントを確認できます。
- 高度分析画面に「寄り付き前チェック」を追加し、日本営業日、前営業日引け後〜当日朝の対象期間、EDINET確認、決算予定データの有無を確認できます。

このアプリは投資学習・リサーチ支援・練習注文用です。実注文、証券会社API接続、利益保証、売買推奨は行いません。

### 開示・決算チェック

「開示・決算チェック」は、選択銘柄について今日または直近に確認すべき材料を整理するリサーチ補助です。表示対象は、決算発表、決算短信、業績修正、配当修正、自社株買い、大量保有報告、有価証券報告書、四半期報告書、臨時報告書、その他重要開示です。

- EDINET: バックエンドの `EDINET_API_KEY` がある場合は、前日引け後〜当日朝の対象期間で EDINET API v2 の提出書類一覧を取得し、証券コードが一致する書類だけを表示します。APIキーがない場合は「APIキー未設定」と表示します。
- TDnet: 規約リスクのあるスクレイピングは行わず、既存の無料RSS・キャッシュ・手動データ相当の材料がある場合のみ表示します。取得できない場合は「データ未取得」と表示します。
- J-Quants: 既存の読み取り専用連携を使い、財務・決算情報の有無を表示します。未設定や該当なしの場合は「未取得」または「データなし」と表示します。
- キャッシュ: 一時保存データを含む可能性がある場合は「キャッシュ利用」と表示します。

EDINETの照合は、まず `secCode` の4桁証券コード一致だけを採用します。`secCode` がない提出書類は、社名の曖昧推定で無理に結び付けず「照合不可」として扱います。大量保有報告、変更報告、公開買付届出書、意見表明報告書、臨時報告書は「確認注意」の材料として表示します。

推奨設定はバックエンド用の `EDINET_API_KEY` です。`VITE_EDINET_API_KEY` はブラウザに渡るため、通常は使わず、ローカル検証やテスト用途に限定してください。実キーは `.env` に設定し、コミットしないでください。

本機能は売買判断ではなく、一次情報確認の抜け漏れを減らすための画面です。表示内容は無料データソースの取得可否や遅延に左右されるため、重要な判断前には必ず公式開示・決算資料を確認してください。

### 寄り付き前チェック

「寄り付き前チェック」は、朝の銘柄確認前に、営業日・祝日・EDINET提出書類・決算予定データの抜け漏れを確認するためのリサーチ補助です。実注文や証券会社APIには接続しません。

- 日本営業日カレンダー: `src/data/japanHolidays.json` の手動祝日データを使います。2026年分を初期登録しています。対象年のデータがない場合は「祝日データ未設定」と表示し、土日と年末年始のみの簡易判定に切り替わります。
- 確認対象期間: 前営業日15:00以降から対象日9:00までを表示します。月曜や祝日明けは直近営業日を前営業日として扱います。
- 決算カレンダー: `/api/research/earnings-calendar` から読み取り専用で取得します。現段階ではJ-Quants決算予定の実取得は未接続で、手動JSONや将来のキャッシュ連携を受けられる土台です。
- 手動データ: `src/data/manualEarningsCalendar.json` に実データを入れた場合のみ「手動データ」と明示して表示します。架空データは実データのように扱いません。
- J-Quants未設定時: 「J-Quants API未設定」「決算予定データは未取得」と表示し、画面は落としません。

この機能は売買推奨ではありません。「確認推奨」「重要予定あり」「一次情報確認」などの注意表示に限定しています。

日本株の寄り付き前デイトレ候補を、統計・ルール・ローカル機械学習による検証補助で絞り込む学習・検証用シミュレーターです。投資助言、買い推奨、証券会社への自動注文は行いません。画面表示は日本語ネイティブ向けに、練習注文・保有銘柄・データ出所・警告を日本語中心で表示します。

## 現在の運用方針

- 固定観察銘柄は `4980.T` デクセリアルズのみです。
- それ以外の候補は、JPX 上場銘柄リストから国内市場の銘柄を読み込み、無料データで一括スクリーニングします。
- `/api/screen` は固定観察銘柄を残したまま、国内市場ユニバースの上位候補を `STOCKS` に反映します。
- Jobs Decision は50万円のシミュレーションとして、指値、売値、損切り、100株単位の株数、想定利幅を表示します。実注文は常にオフです。

## 銘柄ユニバース設定

`.env` で以下を調整できます。

- `ZEN_JPX_UNIVERSE_PATH`: ローカルのJPX銘柄マスタ xlsx/csv を優先使用する場合のパス
- `ZEN_JPX_LISTED_ISSUES_URL`: JPX上場銘柄データURL。標準は月次の全上場銘柄 `.xls`
- `ZEN_JPX_UPDATED_ISSUES_URL`: `.xls` を読み込めない環境で使う更新銘柄 `.xlsx` フォールバック
- `ZEN_SCREEN_MAX_UNIVERSE`: `0` は読み込めた国内市場ユニバースを全件スキャン
- `ZEN_SCREEN_BATCH_SIZE`: yfinance 一括取得のバッチ件数

日本株の寄り付き前デイトレ候補を、統計・ルールベース・検証ログで絞り込む分析支援アプリです。投資助言、買い推奨、証券会社への自動発注は行いません。

## 現在構成

- `backend/server.py` - FastAPI API、候補リスト、ポートフォリオ、詳細分析、ローカルDB
- `backend/preopen_scoring.py` - 寄り付き前に利用可能な情報だけで作る100点スコア
- `backend/advanced_analysis.py` - 履歴検証、ウォークフォワード、モンテカルロ、品質ゲート
- `backend/ml_prediction.py` - scikit-learnを使った無料ローカルML検証補助。候補を強く推奨せず、反証・参考不足・データ抑制を表示
- `backend/daytrade_engine.py` - 板・寄り付き監視用のペーパー検証チケット
- `backend/jquants_bridge.py` - 任意の読み取り専用J-Quants連携
- `src/App.jsx` / `src/index.css` - React UI
- `src/api/apiClient.js` - フロントエンドのAPI接続、バックエンドURL解決、キャッシュ読み書き
- `src/components/` - 候補サマリー、練習ダッシュボード、監視リスト、詳細パネル、保有銘柄、データ出所バッジ
- `src/hooks/useMarketData.js` - 株価、ランキング、詳細、ポートフォリオ、取引履歴、API失敗時のフォールバック表示
- `src/hooks/useSelectedStock.js` - トップ候補、監視リスト、詳細、チャート、練習注文フォームの銘柄同期
- `src/hooks/usePracticeOrder.js` - 練習注文フォーム、入力チェック、練習注文履歴、未約定/約定済み/取消済みの状態管理
- `src/hooks/usePortfolioLedger.js` - 保有銘柄、保有履歴、配分、ポートフォリオ健全性、台帳状態変更の整理
- `src/utils/dataSource.js` - yfinance、Yahooチャート、Stooq、J-Quants遅延、補完データ、一時保存データの表示判定
- `tests/` - スクリーニング、詳細分析、アラート、ポートフォリオ、寄り前スコアの回帰テスト

## 寄り前スコア

`preopen_scoring.py` は当日寄り後の価格、高値、終値、出来高を使いません。入力は完了済み日足と、実運用で別途接続できる任意フィードだけです。未接続のニュース、PTS、寄り前気配、セクター強度は推測で加点せず、UI上では未確認リスクとして表示します。

スコア配分:

- 材料性: 20点
- 出来高急増: 20点
- PTS・寄り前気配: 15点
- テクニカル: 15点
- 地合い・セクター: 10点
- 流動性: 10点
- リスク控除: 最大30点

主なリスク判定:

- 低流動性
- 過熱
- 高ボラティリティ
- 寄り天傾向
- 代替データ使用
- 材料、PTS、気配、地合いフィード未接続

## 起動

### クイック起動 (Windows)

エクスプローラーから以下のスクリプトをダブルクリックするだけで、バックエンドとフロントエンドが自動で起動し、ブラウザで画面が開きます。
- `ZEN_LAUNCHER.bat` (コマンドプロンプト画面が表示されます)
- `ZEN_LAUNCHER.vbs` (バックグラウンドで静かに起動します)

### 手動起動 (コマンドライン)

```powershell
cd C:\Users\BRB33\investment-simulator-pro
python -m pip install -r requirements-daytrade.txt
npm install
python backend/server.py
npm run dev
```

通常の確認URL:

- Frontend: `http://127.0.0.1:5174/`
- Backend: `http://127.0.0.1:8889/api/stocks`
- Pre-open API: `http://127.0.0.1:8889/api/preopen/4980.T`

### 同じWi-Fiのスマートフォンで確認する場合

開発サーバーは `0.0.0.0` で待ち受けるため、同じWi-Fi上のスマートフォンからPCのIPアドレスへ直接アクセスできます。

```powershell
python backend/server.py
npm run dev
```

PCのWi-Fi側IPv4アドレスは次のコマンドで確認できます。

```powershell
Get-NetIPAddress -AddressFamily IPv4 | Where-Object { $_.InterfaceAlias -like '*Wi-Fi*' }
```

スマートフォンで `http://<PC-WIFI-IP>:5174/` を開きます。例: `http://192.168.0.26:5174/`。APIは同じホストの `8889` 番ポートを参照します。CORSはlocalhostとプライベートLANアドレスに限定しています。

### 外出先のスマートフォンで一時確認する場合

一時的に外部から確認する場合は、このPCを起動したままバックエンド、フロントエンド、Cloudflareトンネルを起動します。

```powershell
python backend/server.py
npm run dev
npm run share
```

`npm run share` が表示する `https://*.trycloudflare.com` のURLをスマートフォンで開きます。ブラウザは同じトンネルURLの `/api` を呼び、Viteがローカルバックエンド `127.0.0.1:8889` へ中継します。`8889` 番ポートを直接公開する必要はありません。確認後はトンネルを停止してください。

## 設定

`.env.example` を `.env` にコピーして利用します。APIキーやトークンは空欄のままでも動作します。実キーはコミットしないでください。

主な環境変数:

- `ZEN_API_HOST`, `ZEN_API_PORT`: FastAPIの待受ホストとポート。標準は `127.0.0.1:8889`
- `VITE_ZEN_API_BASE`, `VITE_ZEN_API_PORT`: フロントエンドから見たAPI URLを明示したい場合に使用
- `ZEN_JPX_UNIVERSE_PATH`, `ZEN_JPX_LISTED_ISSUES_URL`, `ZEN_JPX_UPDATED_ISSUES_URL`: JPX銘柄マスタ取得設定
- `ZEN_SCREEN_MAX_UNIVERSE`, `ZEN_SCREEN_BATCH_SIZE`: スクリーニング対象件数と取得バッチ
- `JQUANTS_EMAIL`, `JQUANTS_PASSWORD`, `JQUANTS_REFRESH_TOKEN`: J-Quantsを読み取り専用で使う場合のみ設定。ソースコードへ直書きしないでください。

## データ出所の表示

主要な価格にはデータ出所バッジを表示します。

- `yfinance取得`: yfinanceから取得した日足データ
- `Yahooチャート取得`: Yahoo Finance Japanのチャート取得
- `Stooq取得`: Stooqから取得した代替日足データ
- `J-Quants遅延データ`: J-Quantsの公式遅延データ。リアルタイム価格ではありません
- `補完データ`: 欠損時の推定・補完データ。実際の市場データではないため、投資判断には使わないでください
- `一時保存データ`: ブラウザまたはローカル保存から復元したデータ。最新市場価格と異なる可能性があります
- `出所不明`: データ出所を確認できない参考値

`補完データ` と `一時保存データ` は、画面上に警告文を表示します。補完データの場合、強い買い候補のような表示は抑制し、参考表示またはデータ不足に寄せます。

## データ取得と鮮度管理

市場データ取得は `useMarketData` に分離しています。`App.jsx` は画面全体の接続役として、取得済みデータを候補サマリー、監視リスト、詳細パネル、練習注文、保有台帳へ渡します。

主な取得対象:

- `/api/stocks`: トップ候補と監視リスト用の銘柄
- `/api/portfolio`: 保有台帳と評価額
- `/api/transactions`: 練習注文履歴の再取得
- `/api/market/universe`: 国内市場ユニバースの状態
- `/api/market/rankings`: 条件一致ランキング
- `/api/stock/{ticker}` と `/api/analysis/advanced/{ticker}`: 選択銘柄の詳細と高度分析

APIの一部が失敗しても、画面全体を空にせず、取得できた範囲、ブラウザキャッシュ、デモ用フォールバックの順で表示を維持します。この場合は `一部データ不足`、`参考更新`、データ出所バッジ、警告文で最新性と信頼度を確認できるようにしています。

`useMarketData` は `補完データ`、`一時保存データ`、`出所不明` を検出し、ランキング候補を参考表示へ寄せます。これは新しい投資判断ロジックではなく、実データと誤認しないための安全表示です。J-Quantsは読み取り専用の遅延データとして扱い、リアルタイム価格や板情報としては表示しません。

## AI検証補助

高度分析APIは、選択銘柄の履歴から `mlPrediction` を生成します。これは無料ローカル環境の `scikit-learn` を使う補助機能で、候補を作る主役ではなく、すでに選ばれた候補を疑うための監査役です。

表示する主な内容:

- `参考可`: ウォークフォワード検証が単純基準を上回り、最低限の標本数がある状態
- `参考不足`: 標本数、検証差、データ品質が不足している状態
- `反証あり`: AI検証が単純基準を上回らず、候補を疑う材料がある状態
- `参考表示`: 補完データ、一時保存データ、出所不明など、実市場データとして扱えない状態

AI検証補助は、上昇確率、検証精度、単純基準との差、重要特徴量を表示します。ただし、確率だけで候補順位を上げたり、買い推奨にしたりしません。補完データ、一時保存データ、出所不明データでは強い判断を抑制します。

## 練習注文

練習注文は、手入力した銘柄、価格、株数、メモをシミュレーション用の保有台帳と画面内履歴に残す機能です。証券会社API、実発注、外部RPAには接続しません。

練習注文ステータス:

- `入力中`: フォームを入力・確認している状態
- `未約定`: 練習注文として保存し、まだ約定済み扱いにしていない状態
- `約定済み`: 練習台帳へ保存済みとして扱う状態。実際の市場約定ではありません
- `取消済み`: 練習上の注文を取り消した記録として残す状態
- `データ不足`: 銘柄、価格、株数など、保存に必要な入力が不足している状態
- `参考表示`: 補完データや出所不明データなど、実市場価格として扱えない状態

`補完データ`、`一時保存データ`、`J-Quants遅延データ`、`出所不明` の価格を参照している場合は、フォーム内にも注意を表示します。保存した場合も学習・検証用の参考記録であり、投資判断や実注文には使わないでください。

UIの銘柄選択は `useSelectedStock`、練習注文フォームと履歴は `usePracticeOrder` に分離しています。これにより、監視リストで選んだ銘柄が練習注文フォームとズレないようにしつつ、今後の保有台帳やバックテスト拡張を段階的に行いやすくしています。

画面内のローカル練習履歴は、保存・取消の直後にユーザーが結果を確認するためのセッション内表示です。バックエンド保存に成功した内容は既存APIの保有台帳・取引履歴に従って再取得されます。ページ再読み込み後に残る履歴範囲は、現在のバックエンド保存仕様とブラウザキャッシュの復元仕様に従います。

## 保有台帳と損益確認

保有銘柄、保有履歴、資産配分、ポートフォリオ健全性、売却済み・入力ミス・非表示の台帳状態変更は `usePortfolioLedger` に分離しています。台帳状態変更は既存のローカルAPIへ保存するだけで、証券会社への売却注文や実発注は行いません。

保有銘柄の行を選ぶと、詳細、チャート、練習注文フォーム、保有台帳フォームが同じ銘柄に同期します。売却済み・入力ミス・非表示は、削除ではなく学習用の履歴として残すための状態変更です。

## J-Quants利用時の注意

J-Quants連携は読み取り専用の補助情報です。遅延データであり、リアルタイム板、気配値、約定価格ではありません。未設定でもアプリ本体は動作します。認証情報は `.env` などローカル環境変数で管理し、リポジトリへコミットしないでください。
