# ローカル運用ガイド (Local Operation Guide)

## 🌅 朝のルーチン (最短手順)

朝一番に利用する際は、以下のステップで起動してください。

1. **`npm run up`** を実行して起動を待つ。
2. 接続エラーが出た場合や起動に失敗した場合は **`npm run down`** → **`npm run up`** を実行する。
3. それでもダメなら **`npm run diagnosis`** を実行して、画面の指示に従う。

---

## 運用コマンド

| コマンド | 内容 |
| :--- | :--- |
| **`npm run up`** | **起動**: Backend(健康診断待機) → Frontend(ビルド&待機) → Smoke検証。失敗時は自動クリーンアップ。 |
| **`npm run down`** | **停止**: ポート 3000/8000 の PID 停止と `node`/`python` 名での徹底排除。解放確認付き。 |
| **`npm run diagnosis`** | **診断**: ポート、API、フロントの状態を診断。解決策(NEXT)を画面に表示。 |

---

## 調査とログ (トラブルシューティング)

エラー時は `logs/` フォルダの最新ファイル（時刻付き）を確認してください。

### ログの種類
- **`logs/up_...log`**: 起動中のどこ (`FAILED_STEP`) で止まったか。
- **`logs/down_...log`**: プロセスが正常に終了できたか。
- **`logs/diagnosis_...log`**: ポートや API がどのような通信エラーを返しているか。
- **`logs/backend_startup_...log`**: Python 側の構文エラーやライブラリ不足の詳細。

### 診断コマンド (`npm run diagnosis`) の結論
画面に表示される結論から、以下の対処を行ってください。
- **3000(フロント)未起動**: `npm run up` が途中で止まっていないか確認。
- **8000(バック)未起動**: `logs/backend_startup...` を見て Python エラーを修正。
- **疎通不可**: プロセスはいるが応答がない状態です。`npm run down` で一度リセットしてください。
- **ポート競合**: 別のアプリが 3000/8000 を使っている可能性があります。

---

## 判定ルール (Safety Gate)
`npm run up` 完走時に以下の判定が出ます。
- **OK**: 全データ最新。
- **PARTIAL**: 一部データ欠損（VIXやFX等）。慎重に利用。
- **STALE/FAIL**: データが古すぎる、または致命的エラー。**安全のため自動停止します。**
